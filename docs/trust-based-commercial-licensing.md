# Trust-Based Commercial Licensing

ShakaCode uses Trust-Based Commercial Licensing for commercial and
source-available products.

## Core principle

> Free to learn, evaluate, demo, and use for qualifying open-source projects.
> Paid when the software creates private business value.

## Why trust-based?

We want to reduce adoption friction:

- developers can inspect and evaluate the software
- teams can try it before buying
- qualifying public open-source projects can use it
- honest professional users can understand when payment is required

We do not start with heavy-handed activation, phone-home checks, or intrusive
license enforcement. Product-specific terms still apply.

## Common free uses

Depending on the product-specific license, free use may include:

- learning
- demos
- tutorials
- education
- personal/hobby use
- qualifying open-source use
- limited evaluation
- public reports with attribution

## Common paid uses

Depending on the product-specific license, paid use may include:

- production use
- private repo use
- private CI/CD
- automated PR checks
- coding-agent workflows
- performance-regression gates
- private hosted reports
- paid client work
- hosted/managed services built on the product

## Product-specific terms control

Each ShakaCode product has its own binding license file. This document is a
shared philosophy and template source, not a substitute for product license
terms.
