# Trust-Based Commercial Licensing FAQ

## Is this legal advice?

No. This repository describes ShakaCode's shared licensing philosophy and
reusable template language. Product-specific license files control.

## Is source-available software the same as open source?

No. Do not call a product open source unless the applicable product license is
an OSI-approved open-source license at that time.

## What is generally free?

Product-specific terms control, but common free uses may include learning,
evaluation, demos, tutorials, education, personal use, qualifying public
open-source projects, public reports with attribution, and limited trials.

## What generally requires payment?

Product-specific terms control, but common paid uses may include production
use, private CI/CD, automated PR checks, coding-agent workflows, private hosted
reports, paid client work, and hosted or managed services built on a ShakaCode
commercial product.

## Can teams inspect the source before buying?

For source-available products, yes. The goal is to let teams understand,
evaluate, and adopt with low friction while still requiring a commercial
license when the software creates private business value.

## Can coding agents use ShakaCode commercial products?

Agents may use products according to the product documentation and license. They
should install or invoke the product as a dependency, tool, library, CLI, or
service. Agents should not copy, vendor, translate, port, or reimplement
substantial commercial source to avoid licensing.

## Which license wins if this repo and a product repo differ?

The binding license file in the product repository controls.
